Release Notes for Freeze 02.05.2004

  • The depth phase handling in MKTIM4P and MKX4P was not correct. wrong handling of take of angles at the surface produced an error which could be as large as 3s for deep sources. This is repaired.
  • A new Section on andvanced tasks has been added to the Documentation. It tries to explain how to make efficient use of TTBOX routines.
  • The calling sequence of MKRAYDEPTH has been changed: he planet's radius has been added as input parameter for use in the flat earth transfom.
  • The calling sequence of MKTTBOXVALID has been extended: two optional parameters, takeabs and radius, have been added to allow for more flexible testing and plotting. The old calling sequence is still valid, some defaults for the new parameters are used then which reproduce the results of the old version.
