Seismic Phase Names used in TTBOX

The current version of TTBOX handles different phases by special phase specific code which is located in MKX4P, MKTIM4P, and MKSMARTTAKEOFF. This is not very elegant, but easy to implement. MKPHASECATALOG tells you which phases are available.
Future versions, hopefully, will have a more versatile phase name parser.

An exception are depth phases, which are handled recursively by combining an upgoint p or s ray with a standard phase for a surface focus. So every phase TBOX can handle is available as depth phase automatically.

The nomenclature used is a simplified standard nomenclature. The following table summarizes the available symbols and their meaning.

Symbol Explanation
P P wave leg
S S wave leg
p Depth phases: the P leg which goes upward from the source.
s Depth phases: the S leg which goes upward from the source.
K Outer core P wave leg
I Inner core P wave leg
c Reflection at Core Mantle Boundary
i Reflection at Inner Core Boundary

Multiples of all phases can be handled: If you seismic phase is named X, then Xn denotes the n-th multiple of this phase. For example, SKS2 is the second multiple SKSSKS. An arbitrary integer number can be appended to all phase names TTBOX recognizes.

Limitations
  • Diffracted waves and head waves are currently not supported.
  • Branch designators for travel time triplications (like "ab", "bc", "df" for PKP) are currently not supported.
  • Reflections at arbitrary discontinuities are supprted only in the total reflection range.
  • Conversions at arbitrary discontinuities are currently not supported.
  • Multiple phase handling does currently not encompass phases like PK5P (PKKKKKP), numbers are allowed only at the end of a phase name.
